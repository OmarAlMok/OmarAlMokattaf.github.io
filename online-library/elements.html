About the Online Library and How to Run

Your guide to maneuver through the Online Library and exploit its unique functions

Your Guide to understand the Online Library

The Online Library Program is a desktop application built with:

  • PyQt6: Graphical User Interface (GUI)
  • Flask: Backend server to handle HTTP requests
  • JSON: Data storage for books and user information
  • Multiprocessing: Runs Flask and the GUI simultaneously

The program lets you:

  • Add books
  • Delete books
  • Borrow and return books with a due date and a borrower's name
  • View a book's metadata by double-clicking in the GUI
  • Search books if the inputted name is an exact match
  • Filter books by category and availability

Installation

  • To install and run the program, make sure you have python installed. Then, install PyGt6 and Flask.

How to run

  • run main.py

How the program starts:

  • When you run main.py, the flask server gets activated at http://localhost:5000. Then, after 2 seconds, the GUI is activated. When the GUI is closed the server terminates.

Architecture

  • The user has acces to the GUI.
  • The GUI sends HTTP requests to the Flask backend server.
  • The Flask server processes the requests, interacts with the JSON data storage, and sends responses back to the GUI.

Structure

  • main.py: Starts the backend and the GUI.
  • gui.py: Handles the GUI.
  • backend.py: Flask server managing database operations.
  • data.db: JSON database storing all book information
  • README.md: Documentation file

HTTP Endpoints

  • GET /books: Retrieve all book
  • POST /books: Add a new book
  • DELETE /books/id: Delete a book
  • POST /books/id/borrow: Borrow a book
  • POST /books/id/return: Return a book
  • GET /books/id: Get information about a specific book

Limitations

  • The program must run locally
  • No user authentication implemented
  • Basic error handling
  • data.db is stored as plain JSON and is not encrypted.
  • Any actions that take place in the GUI takes 2-4 seconds to be uploaded
